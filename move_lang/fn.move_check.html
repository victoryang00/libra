pub fn move_check(
    targets: &[String], 
    deps: &[String], 
    sender_opt: Option<Address>, 
    interface_files_dir_opt: Option<String>, 
    sources_shadow_deps: bool
) -> Result<(FilesSourceText, Result<(), Errors>)>

Given a set of targets and a set of dependencies

  • Checks the targets with the dependencies (targets can be dependencies of other targets)
  • If the sources_shadow_deps flag is set, modules defined in targets can shadow (i.e., override) the modules in dependencies, and the checker will not complain about it. In other words, if the same module id (address::name) is found in both targets and dependencies, the module definition in targets takes priority. Does not run compile to Move bytecode Very large programs might fail on compilation even though they have been checked due to size limitations of the Move bytecode
